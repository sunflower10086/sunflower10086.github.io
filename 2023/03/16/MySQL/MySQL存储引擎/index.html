2.存储引擎

存储引擎就是存储数据、建立索引、更新/查询数据等技术的实现方式。存储引擎是基于表的，而不是基于库的，所以存储引擎可以被称为表类型

使用方法

innoDB

innoDB兼顾高可靠和高性能

特点

  • DML操作CRUD，遵循ACID，支持事务
  • 行级锁，提高并发访问性能
  • 支持外键

Memory

他很特殊，数据存放在内存中，一旦断电数据就会消失不见，因此这些表只能作为临时表或缓存使用

特点

  • 内存存放
  • hash索引
