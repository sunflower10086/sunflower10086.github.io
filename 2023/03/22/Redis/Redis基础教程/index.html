Redis基础

Redis的一些基础数据类型和使用方法

1.Redis数据结构介绍

keys一般都是String，keys的类型十分丰富

value 例子
String hello world
Hash {name: “jack”, age: 21}
List {A -> B -> C -> C}
Set {A, B, C}
SortedSet {A: 1, B: 2, C: 3}
GEO {A: (120.3, 30.5)}
BitMap 011011011011011011
HyperLog 011011011011011011

操作方式可以去官网学习，官网讲的更详细

2. Redis的通用命令

1.keys

查找key，可以有通配符*, ?(*表示多个，?表示一个字符)

keys *效率太差，又因为redis是单线程的，所以不建议我们在生产环境设备上使用

2.del

删除一个指定的key

3.exists

判断一个key是否存在，返回值是1或0

4.expire（十分常用）

给一个key设置有效期，到期时key会被自动删除

5.TTL

查看一个key的有效期，-2表示已经过期，-1表示永不过期

3. String类型

Redis中最简单的存储类型

value时字符串，根据字符串格式不同，分为三类

  • string：普通字符串
  • int：整数类型，可以做自增，自减操作
  • float：浮点类型，可以做自增，自减操作

底层都是字节数组存储，只不过编码方式不同，最大空间不超过512MB

3.1 String相关命令

  • set：添加或修改一个已经存在的String类型的键值对，可接的参数十分多，命令行会自动提示
  • get：根据key获取String类型的value
  • mset：批量添加多个String类型的键值对
  • mget：根据多个key获取多个String类型的value
  • incr：让一个整形的key自增1返回增长后的数值
  • incrby：让一个整形的key自增并指定步长，比如incrby age 2返回增长后的数值
  • incrbyfloat：让一个浮点类型的数字自增并指定步长
  • setnx：添加一个String类型的键值对，前提是这个key不存在**(失败返回0，成功返回1)**
  • setex：添加一个String类型的键值对，并指定有效时间

5. Hash类型

Hash类型存储结构

5.1 Hash常见命令

  • hset key field value: 添加或修改hash类型key的field的值

  • hget key field：获取一个hash类型key的field的值（例子同上）
  • hmset：批量添加多个hash类型的key的field的值

  • hmget：批量获取多个hash类型key的field的值

  • hgetall：获取一个hash类型中的key中的所有的field和value值

  • hkeys：获取一个hash类型的key中的所有的field
  • hvals：获取一个hash类型的key中的所有的value

  • hincrby：让一个hash类型key的字段值自增指定步长（自增就加正数，自减就加负数）

  • hsetnx：添加一个hash类型的key的field值，前提是这个field不存在，否则不执行

6. List类型

在redis中可以把List类型用作栈，队列，阻塞队列

list命令以l开头代表在队列左侧执行，r开头代表在队列右侧执行

6.1 List常见命令

  • lpush key element ...：向列表左侧插入一个或多个元素
  • lpop key：移除并返回列表左侧的第一个元素，没有则返回nil
  • rpush key element ...: …
  • rpop key：…
  • lrange key star end：返回一段角标范围内的所有元素（范围为0 ~ -1时返回所有）
  • blpop 和 brpop：与lpop和rpop类似，在没有元素时等待指定时间，而不是直接返回nil

7. Set类型

set类型特点

1.无序

2.元素不重复

3.查找快

4.支持交、并、差集运算

7.1 Set常见命令

  • sadd key member ...：向set中添加一个或多个元素
  • srem key member ...：移除set中的指定元素
  • scard key：返回set中元素的个数
  • sinmember key member：判断一个元素是否存在于set中
  • smembers：获取set中的所有元素
  • sinter key1 key2：求key1于key2的交集
  • sdiff key1 key2：求key1与key2的差集
  • sunion key1 key2：求并集

8. SortedSet

SortedSet是一个可排序的Set集合，SortedSet中的每一个元素都带有一个score属性，可以基于score进行排序

特点

1.可排序

2.元素不重复

3.查询速度快

经常被用来实现排行榜这样的功能

8.1 SortedSet常见命令

  • zadd key score member：添加一个或多个元素到sortedset，如果member已经存在，那么就更新score值
  • zrem key member：删除sortedset中的一个指定元素
  • zscore key member：获取sortedset中的执行元素score值
  • zrank key member：获取指定元素的排名（默认升序）
  • zcard key：获取元素个数
  • zcount key min max：统计score值在给定范围内的所有元素的个数
  • zincrby key increment mmember：自增，步长为指定的increment值
  • zrange key min max：按照score排序之后，获取指定排名范围内的元素（排名从0开始）
  • zrangebyscore key min max：按照score排序后，获取指定score范围内的元素
  • zdiff、zinter。zunion：差集、交集、并集、

所有的排名默认都是升序，如果要降序则在命令的z后面添加rev即可
