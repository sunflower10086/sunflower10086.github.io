树的重心

重心是指树中的一个结点，如果将这个点删除后，剩余各个连通块中点数的最大值最小，那么这个节点被称为树的重心。

当我们找的不是根节点的时候比如上图中的4，此时有三个连通块，下面的连通块递归算，上面的连通块就是总数减去下面的个数，然后我们找到连通块中的最大值，然后在每个最大值中找到最小值，此时那个最小值的的删除的节点就是树的重心

思路

Dijkstra 的整体思路比较清晰
即进行n（n为n的个数）次迭代去确定每个点到起点的最小值 最后输出的终点的即为我们要找的最短路的距离

所以按照这个思路除了存储图外我们还需要存储两个量

每次迭代的过程中我们都先找到当前未确定的最短距离的点中距离最短的点
（至于为什么是这样那么这就涉及到Dijkstra算法的具体数学证明了 有兴趣的同学可以百度一下）

通过上述操作当前我们的t代表就是剩余未确定最短路的点中 路径最短的点
而与此同时该点的最短路径也已经确定我们将该点标记

然后用这个去更新其余未确定点的最短距离

进行n次迭代后最后就可以确定每个点的最短距离
然后再根据题意输出相应的 要求的最短距离

bellman-ford

两重循环，第一层遍历所有点，第二层遍历所有边，如果给了我们只能使用k条边，那么第一层就为k

backup数组的作用，只用上一层更新过的边去更新这一层，否则的话一个被更新完的边就会把所有边都更新掉

比如下面这个例子，刚开始我们的dist数组（最多使用一条边）

dist[1] = 0, dist[2] = 正无穷, dist[3] = 正无穷

先遍历了1到2的边，那么

dist[1] = 0, dist[2] = 1 dist[3] = 正无穷

如果此时不用上一层的距离，

dist[3] = dist[2] + 1 = 2;

但是只用一条边的dist[3] = 4;

如果我们使用了上一层的距离即（dist[1] = 0, dist[2] = 正无穷, dist[3] = 正无穷）

dist[3] = dist[1] + 4 = 4;

spfa

spfa求最短路

spfa实际上是对bellman-ford算法的一个优化，在bellman-ford算法中，我们的松弛操作

其实是对每个边都进行更新，但是 没必要，只有当backup[a]变小的时候我们的dist[b]才会变小，所以我们用一个队列去存储所有变小的边，然后通过这个变小的边去更新他的后继节点，时间复杂度基本为O(m)，最坏情况下为O(nm)，和dijkatra算法十分相似

spfa判断负环

讲解

根据抽屉原理，我们一共只有 n 个点，如果没有负环，那么每次求最短路，经过一个点的边数绝对不会超过 n ，如果超过了n则说说明我们使用了 n + 1 个点，但是我们只有n个点，则说明出现了负环

我们只需要再开一个cnt数组，在每次更新边的时候 cnt[j] = cnt[t] + 1，进行一下判断即可。

因为我i们是在判断是否有负环，没有在求最短路，所以要把所有的边在初始化的时候全部放到队列中去
